NAME

ipc_shm_open - Open a named shared memory region

 SYNOPSIS

#include "ipc.h"
int ipc_shm_open(const char *name, void **addr, unsigned int *size);

 DESCRIPTION

The ipc_shm_open() function provides access to a named shared memory region created by any process in the system using ipc_shm_create().
The ipc_shm_open() and ipc_shm_create() functions work with names that exist within a unique namespace, other functions in ipc.h do not share this namespace.
The name argument indicates the name of the shared memory region, it must point to a string consisting of alphanumeric characters and/or the special characters '-' and '_', other characters are illegal.
The addr argument is a pointer to a void pointer which the address of the shared memory region is to be written to.
The size argument is a pointer to an unsigned integer, upon successful completion of the function the value at size will indicate the size of the shared memory region.

 RETURN VALUE

Upon successful completion the function returns zero.

 ERRORS

In case of an error one of the following values will be returned:
[IPC_ERROR_NULL_POINTER_ERROR]
The name, addr or size arguments are NULL.
[IPC_ERROR_INITIALIZATION_ERROR]
The length of the string pointed to by name exceeeds IPC_NAME_MAX_LENGTH.
or
There is an illegal character present in the string pointed to by name.
or
An shared memory region with this name does not exist.
[IPC_ERROR_MALLOC_ERROR]
A call to malloc() has failed due to insufficient memory.
[IPC_ERROR_SYSTEM_CALL_ERROR]
The function has failed due to an operating system function error.

 EXAMPLES

See the documentation for ipc_shm_create() for an example of ipc_shm_open().
