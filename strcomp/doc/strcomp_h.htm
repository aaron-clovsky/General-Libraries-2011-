NAME

strcomp.h - String comparison library

 SYNOPSIS

#include "strcomp.h"

COMPATIBILITY

ANSI C

 DESCRIPTION

The strcomp.h header file provides string comparison functions.
The following functions are defined: 
int   strcomp_lcs(const char *str1, const char *str2);
float strcomp_lcs_normalized(const char *str1, const char *str2);
int   strcomp_levensthein(const char *str1, const char *str2);
float strcomp_levensthein_normalized(const char *str1, const char *str2);

COMPILE TIME OPTIONS

The following constants are used by this library and may be overridden at compile time:
[STRCOMP_FIXED_BUFFER]
Default Value: 0

Defining this value as non-zero causes functions normally requiring dynamic allocation
(such as strcomp_levensthein_*) to use pre-allocated memory, the size of this allocation is
defined by STRCOMP_FIXED_BUFFER. This may provide a sizeable performance increase when a
large number of comparisons must be performed. Should the required memory for the comparison
exceed STRCOMP_FIXED_BUFFER then the function will return STRCOMP_ERROR.
When STRCOMP_FIXED_BUFFER is non-zero some functions will no longer be thread-safe.

NOTES

This library is thread safe when STRCOMP_FIXED_BUFFER is zero.
