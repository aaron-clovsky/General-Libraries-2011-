NAME

ipc_semaphore_open - Open a named semaphore

 SYNOPSIS

#include "ipc.h"
int ipc_semaphore_open(const char *name, ipc_semaphore *semaphore);

 DESCRIPTION

The ipc_semaphore_open() function opens a named semaphore created by any process on the system using ipc_semaphore_create().
The ipc_semaphore_open() and ipc_semaphore_create() functions work with names that exist within a unique namespace, other functions in ipc.h do not share this namespace.
The name argument indicates the name of the semaphore, it must point to a string consisting of alphanumeric characters and/or the special characters '-' and '_', other characters are illegal.
The semaphore argument must point to an uninitialized ipc_semaphore type variable.
The semaphore opened by ipc_semaphore_open() may be waited on/decremented with the ipc_semaphore_wait(), ipc_semaphore_trywait() or ipc_semaphore_timedwait() functions.

The semaphore may be incremented with the ipc_semaphore_post() function.

Once the semaphore is no longer in use the semaphore should be closed by calling ipc_semaphore_close().
The semaphore will cease to exist when all processes that have opened the semaphore have either terminated or called ipc_semaphore_close().

 RETURN VALUE

Upon successful completion the function returns zero.

 ERRORS

In case of an error one of the following values will be returned:
[IPC_ERROR_NULL_POINTER_ERROR]
The name or semaphore arguments are NULL.
[IPC_ERROR_INITIALIZATION_ERROR]
The length of the string pointed to by name exceeeds IPC_NAME_MAX_LENGTH.
or
There is an illegal character present in the string pointed to by name.
or
A semaphore with this name is not currently opened by a running process on the system.
[IPC_ERROR_MALLOC_ERROR]
A call to malloc() has failed due to insufficient memory.
[IPC_ERROR_SYSTEM_CALL_ERROR]
The function has failed due to an operating system function error.

 EXAMPLES

See the documentation for ipc_semaphore_create() for an example of ipc_semaphore_open().
